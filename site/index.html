Never. TicketLens is local-first. All data fetched from Jira stays on your machine in ~/.ticketlens/cache/. Nothing is sent to external servers. The metadata-only privacy model means your ticket content never leaves your environment.
Yes. TicketLens supports Jira Cloud (Basic auth), Jira Server/DC 8.14+ (Bearer PAT), and older Jira Server instances using username+password auth. Run ticketlens init — it auto-detects your instance type and guides you through setup.
The /jtb skill works natively inside Claude Code. The CLI output is structured for any AI tool — paste it into Cursor, Copilot Chat, Windsurf, or any LLM context window. The plain-text, structured output is designed to be AI-ready out of the box.
Pro is a per-seat license key delivered via LemonSqueezy. Run ticketlens license activate YOUR_KEY and Pro features unlock immediately — no restart required. Your license is bound to your machine and can be transferred via ticketlens license deactivate.
Pro is for individual developers: AI-powered ticket summaries, unlimited compliance checks, depth-2 traversal, and a scheduled daily triage digest. Team adds cross-team visibility: filter by assignee and sprint, a live team triage dashboard, Slack/Teams alerts, and CSV/JSON export. Both include everything in Free.
